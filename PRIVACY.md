# AskInPage 隐私政策

生效日期：2026 年 7 月 21 日

AskInPage 是一个由用户主动选择网页文字后，调用用户自行配置的模型服务进行解释或翻译的浏览器扩展。本政策说明扩展会处理哪些数据、数据的用途以及存储方式。

## 处理的数据

当用户主动发起解释、翻译或追加提问时，AskInPage 会处理并向用户配置的模型服务发送以下数据：

- 用户选中的文字；
- 与选中文字相关的网页上下文；
- 页面标题和页面地址；
- 用户输入的追加问题；
- 用户配置的回答格式要求。

开启“快速模式”后，发送给模型服务的网页数据仅包含当前段落和选中的文字。

AskInPage 只会在用户主动触发解释或翻译功能时发送上述数据。AskInPage 开发者不运营模型中转服务器，不会接收、记录或保存上述网页内容和问题。数据会直接发送至用户在扩展设置中配置的模型服务，并受该模型服务提供方的隐私政策和数据处理条款约束。

## API Key

用户填写的 API Key 仅保存在当前设备的浏览器扩展本地存储中，不会通过浏览器同步功能同步到其他设备，也不会发送给 AskInPage 开发者。API Key 只会发送至用户配置的模型服务，用于完成身份验证。

如果用户主动使用“导出 JSON”功能，API Key 会包含在导出的配置文件中。导出文件由用户自行保管，AskInPage 不会自动上传该文件。

## 同步设置

不包含 API Key 的模型连接信息、界面偏好、快捷键和其他插件设置可能通过浏览器的扩展同步存储功能同步到用户登录的其他浏览器实例。

## 数据安全

远程模型服务地址必须使用 HTTPS。本机回环地址 `localhost`、`127.0.0.1` 和 `::1` 可以使用 HTTP，以兼容用户在本机运行的模型服务。

## 用户控制

用户可以随时：

- 在插件弹窗中停用 AskInPage；
- 在设置页面修改或删除 API Key；
- 删除模型服务配置；
- 卸载扩展，以清除扩展在当前设备保存的数据。

## 权限用途

AskInPage 使用网页访问权限在用户访问的页面中检测主动选择的文字、提取必要上下文并显示操作界面。扩展不会在用户未主动触发解释或翻译时将网页内容发送给模型服务。

AskInPage 使用存储权限保存模型连接配置、界面偏好、快捷键和 API Key。API Key 使用仅限当前设备的本地存储。

## 政策更新

如果 AskInPage 的数据处理方式发生变化，本政策会随扩展版本同步更新，并在必要时向用户提供醒目说明。
