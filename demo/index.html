Tokenization：把文字变成数字

大型语言模型（Large Language Model，LLM）并不直接处理文字，而是把输入切分成 token 后再处理。Token 是模型认知世界的最小单位，可以是一个字、半个词、一个标点，甚至一个空格。

以 GPT 系列模型使用的 BPE（Byte Pair Encoding）算法为例：算法从字符级别开始，反复合并出现频率最高的相邻字节对，直到词表达到预设大小（通常 32K–200K token）。"tokenization" 这个词可能被切成 ["token", "ization"]，而低频词则可能被切成逐字节的序列。

这意味着同一段话用中文写和用英文写，消耗的 token 数可能差异巨大。中文字符在 BPE 词表中通常以 2–4 个字节表示，因此相同信息量的中文往往比英文消耗更多 token——这直接影响 API 调用成本和上下文窗口的利用率。

Self-Attention：每个词都在看其他词

Transformer 架构的核心是 self-attention（自注意力机制）。处理一个句子时，模型不像 RNN 那样顺序读取，而是让每个位置的 token 同时"注意"序列中所有其他 token，计算出对每个位置应该分配多少权重。

数学上，attention 的核心公式是：

Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V

其中 Q（Query）、K（Key）、V（Value）是对输入做线性变换后得到的矩阵，dₖ 是 key 的维度。直觉上：Q 是"我想找什么"，K 是"我能提供什么索引"，V 是"实际内容"。Multi-Head Attention 把这个过程并行做多次，每个 head 关注不同的语义子空间，最后拼接结果。

In-Context Learning 与 Prompt Engineering

GPT-3 在 2020 年展示了惊人能力：few-shot learning——给模型几个示例，它就能在不更新任何参数的情况下学会新任务。这不是传统意义上的学习，而是模型在 forward pass 里完成了类似梯度下降的信息处理。

这让 prompt engineering（提示词工程）成为一门有价值的技术。Chain-of-Thought（CoT）提示通过在示例中加入推理步骤，可以大幅提升模型在数学和逻辑任务上的表现。研究者发现，仅仅在 prompt 里加上 "Let's think step by step" 就能显著提升准确率——这被称为 zero-shot CoT。

Emergent Abilities：规模带来的惊喜

研究者观察到，某些能力会在模型规模超过某个阈值后突然出现，而不是随参数量线性增长。这种现象被称为 emergent ability（涌现能力），常见于算术推理、多步逻辑和跨语言迁移等任务。

"Perhaps the most surprising emergent behaviors are those that were not anticipated before they were observed." — Anthropic, 2022

不过，近期也有研究者指出，涌现现象可能部分是 benchmark 选取和评估指标设计的产物——换用不同粒度的评估，有些"涌现"曲线会变成连续的 S 型曲线。

Hallucination 与 RAG

LLM 最广为人知的缺陷之一是 hallucination（幻觉）：模型会以极高的置信度生成看似合理但实际错误的内容。这源于模型的本质——它是在预测最可能的 token，而不是在"查找事实"。

RAG（Retrieval-Augmented Generation，检索增强生成）是目前最主流的缓解方案。核心思路是：在生成答案之前，先用向量检索从知识库里取出相关片段，再把片段塞进 prompt 作为上下文。

一套完整的 RAG 管道通常包含：

  • Embedding 模型：把文本转成高维向量（如 1536 维），捕获语义信息
  • Vector Store：存储向量，用 cosine similarity 或点积做相似度搜索（常见选型：Qdrant、Weaviate、Pinecone）
  • Reranker：对检索结果精排，用 cross-encoder 过滤低相关性内容，提升召回质量
